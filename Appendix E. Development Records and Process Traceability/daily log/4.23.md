阶段：摄像头bring-up(通信与控制链路验证)

今日目标
    在STM32H743平台上完成摄像头模块（OV2640）的底层通信验证，具体包括：
        1.打通I2C总线并确认板载设备
        2.验证PCF8574 IO 扩展器可控性
        3.通过PCF8574控制摄像头PWDN引脚
        4.建立SCCB（摄像头控制总线）通信
        5.尝试读取摄像头ID，验证模块是否成功上电并通信

完成内容
    1.I2C总线验证
        成功初始化I2C2(PH4/PH5)
        编写I2C扫描函数并通过串口输出结果
        扫描到多个设备地址：
            0x10, 0x1E, 0x20, 0x50, 0x6A, 0x76, 0x7E
    其中：
        0x20 确认为 PCF8574（IO 扩展器）
    说明I2C总线工作正常，进入下一阶段

    PCF8574控制验证
        实现PCF8574_Write()函数
        通过写入不同数据验证IO输出变化
        初始化使用0x00时触发蜂鸣器，确认IO输出真实生效
        优化为“单bit控制”，避免干扰其他外设
    最终验证：
        PCF8574 write 0xFF OK 
        PCF8574 write 0xFB OK 
        PCF8574 write 0xFF OK
    结论：
        PCF8574可被MCU稳定控制，IO输出链路完全打通
    
    摄像头PWDN控制
        确认PCF8574的P2引脚连接DCMI_PWDN
        通过写0xFB实现PWDN拉低(摄像头工作状态)
    建立链路：
        STM32 → I2C → PCF8574 → P2 → 摄像头 PWDN
    
    SCCB(摄像头控制接口)建立
        使用PB3/PB4模拟SCCB(软件I2C)
        修正GPIO模式为Open-Drain
        实现最小SCCB通信函数：
            Start / Stop
            Write Byte / Read Byte
            Read Register

    摄像头ID读取(关键突破)
        在以下条件下：
            PWDN已释放
            RESET已正确控制
            SCCB时序正常
        成功读取寄存器：
            Reg 0x0A = 0x26 
            Reg 0x0B = 0x42 
            Reg 0x1C = 0x7F 
            Reg 0x1D = 0xA2
        关键识别
            0x0A / 0x0B → 0x2642
        结论：
            成功识别OV2642摄像头，通信链路完全打通
    
关键问题与解决
    串口乱码问题
        原因：浏览器串口工具不稳定
        解决：更换为XCOM，问题消失
    
    I2C扫描结果异常
        原因：输出被串口工具吞掉
        解决：增加延时 + 更换工具
    
    蜂鸣器误触发
        原因：PCF8574全字节写入影响P0(BEEP)
        解决：改为bit级控制
