日期：4.29

背景与问题来源
    在前日基于串口调试(UART)的工程中，已完成部分摄像头集成工作，但存在一个未解决的关键问题：
        VSYNC/HREF输出异常(不正确或不可用)
    该问题在当时：
        未查明原因
    
    原串口调试方式有重大缺陷：
        原工程完全依赖串口打印调试：
            修改代码 → 编译 → 烧录 → 打开串口工具 → 观察输出
        在简单模块中尚可接受，但在摄像头系统中暴露出严重缺陷：
            信息粒度不足
                无法观察GPIO实时电平
                无法观察寄存器状态
                无法捕捉时序变化(VSYNC/PCLK/HREF)
            调试效率低
                每次修改必须重新烧录
                无法快速试错
            系统复杂度过高
                要排查VSYNC/HREF异常，需要同时观察：
                    GPIO状态 + SCCB寄存器 + 控制链路 + 时序
            串口调试完全无法胜任
        
    关键决策原因
        在此基础上，形成如下判断：
            1.当前工程存在未解决的底层问题(VSYNC/HREF异常)
            2.串口调试无法有效定位该问题
            3.将原工程改造为Debug模式调试：
                需要大规模重构
                复杂度接近重写
        因此：
            改造成本 >= 重构成本
        最终决定：
            放弃原工程
            全部推倒重来
            基于Debug模式重建调试体系

新调试体系设计
    核心转变：
        UART打印 → Debug变量驱动
    即：
        “输出日志” → “可观测状态机”
    
    状态推进
        dbg_step
    通信链路：
        dbg_pcf_ok
        dbg_pcf_write_ok
        dbg_hal_status
        dbg_hal_error
    摄像头识别：
        dbg_ov_pid / dbg_ov_ver / dbg_ov_id_ok
    图像输出信号
        dbg_pclk_cnt / dbg_vsync_cnt / dbg_href_cnt

调试过程
    1.I2C与PCF8574验证
        结果：
            I2C通信正常
            PCF8574可读写
            IO控制有效
    
    2.摄像头控制链路
        在该步骤中，参考周五工作日志，确认：
            PCF8574 P2 引脚连接摄像头 PWDN
        即：
            CAMERA_PWDN_BIT = 2
        控制逻辑：
            bit2 = 0 → 摄像头工作
            bit2 = 1 → 摄像头关机
        建立完整链路：
            STM32 → I2C → PCF8574 → PWDN → OV2640
    
    3.SCCB 通信与摄像头识别
        实现SCCB(软件I2C)，读取寄存器：
            PID = 0x26
            VER = 0x42
        结论：
            摄像头识别成功
            通信链路完全打通
    
    4.VSYNC/HREF/PCLK问题复现与定位
        在新工程初期：
            PCLK / VSYNC / HREF 全部无输出
        此现象与旧工程问题形成呼应：
            旧工程：信号异常（未查明）
            新工程：信号完全无输出
        通过Debug变量直接观察GPIO状态，快速定位问题性质：
            摄像头未进入图像输出模式

    5.引入开源初始化方案
        参考Github开源工程：
            https://github.com/SimpleMethod/STM32-OV2640/tree/master
        提取核心初始化流程：
            OV2640_JPEG_INIT
            OV2640_YUV422
            OV2640_JPEG
            分辨率配置（320x240）
        移植后验证：
            dbg_pclk_cnt  = 82141
            dbg_vsync_cnt = 117
            dbg_href_cnt  = 469
        结论：
            摄像头成功输出DVP图像信号
            PCLK / VSYNC / HREF 全部正常
