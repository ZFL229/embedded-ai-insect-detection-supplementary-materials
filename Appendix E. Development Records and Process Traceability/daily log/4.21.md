工作内容概述：
    进入完成第六阶段前置验证任务之一：
        按键单次触发模型验证(KEY → 单次事件)
    将STM32H743平台上，基于已有GPIO配置，完成按键输入模块的设计与验证，实现了将机械按键从“不稳定物理输入”抽象为“稳定软件事件”的目标

实现内容
    1.硬件资源选取
        按键：
            KEY0 → PH3（低电平按下）
        LED：
            LED1 → PB0（低电平点亮）
        按键采用内部上拉输入模式：
            松开：高电平  
            按下：低电平
    
    2.软件结构设计
        为避免主函数过度复杂，本次实现延续模块化设计，将功能拆分为：
            bsp_key.c / .h   → 按键输入与事件检测
            bsp_led.c / .h   → LED 控制
            main.c           → 调度与逻辑调用

    3.按键事件模型设计
        未采用”按下即触发“的简单电平判断方式，而是引入：
            状态机 + 软件去抖
        构建如下四状态模型：
            KEY_IDLE
            → KEY_DEBOUNCE_PRESS
            → KEY_PRESSED
            → KEY_DEBOUNCE_RELEASE
            → KEY_IDLE
        仅在完成：
            按下 → 松开
        完整闭环后，产生一次有效事件

    4.主循环逻辑
        主循环中周期性调用：
            if (BSP_KEY_ScanEvent() == 1)
            {
                BSP_LED_Toggle();
            }
        实现：
            一次按键 → 一次LED翻转

验证过程
    针对按键输入稳定性，设计并完成如下四类测试：
        1.单机测试
            操作：单次按下并松开
            结果：LED状态翻转一次
        2.长按测试
            操作：持续按住按键
            结果：LED不发生连续翻转
        3.连续点击测试
            操作：快速连续点击5~10次
            结果：每次点击均触发一次翻转，无丢失或反复
        4.静置测试
            操作：不触碰按键
            结果：LED状态稳定，无误触发

关键结论：
    本阶段验证结果表明：
        机械按键输入已成功抽象为稳定事件源
    具体表现为：
        一次完整按压动作 → 精确对应一次软件事件
        消除了按键抖动带来的重复触发问题
        可作为后续系统触发机制的可靠输入口
