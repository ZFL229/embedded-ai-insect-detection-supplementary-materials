2.STM32CubeIDE串口与时钟配置
    USART配置
        启用USART1（Asynchronous 模式）
        引脚配置：
            PA9 → TX
            PA10 → RX
        参数设置：
            波特率：115200
            数据位：8位
            校验：None
            停止位：1
    
    时钟配置问题分析
        初始现象：
            使用HSI作为系统时钟树时：
                CubeIDE提示Clock Warning
                USART时钟路径标红
        进一步分析：
            该Warning并非非法配置，而是：
                时钟树未完全配置（PLL等未参与）
                尝试切换USART时钟源为CSI后：
                    Warning消失
                    系统可正常生成代码
            结论：
                当前阶段可接受简化时钟配置，仅保证UART可稳定工作，无需提前完成完整时钟树设计。
    
    串口发送功能验证
        实现最小测试代码：
            HAL_UART_Transmit(&huart1, msg, sizeof(msg)-1, HAL_MAX_DELAY);
            HAL_Delay(1000);
        实验结果：
            串口助手成功收到周期性字符串：
                H7 UART test
        验证结论：
            串口发送链路正常
            时钟配置满足当前需求

    串口接收与回显验证
        实现单字节回显：
            HAL_UART_Receive(&huart1, &rx, 1, HAL_MAX_DELAY);
            HAL_UART_Transmit(&huart1, &rx, 1, HAL_MAX_DELAY);
        实验结果：
            PC发送字符可被MCU接收并回传
            双向通信链路成立
        ASCII/HEX数据现象分析
            问题现象：
                串口助手出现"？"字符
            分析过程：
                切换为HEX模式后观察到：
                0x0D, 0xFF
            结论：
                串口本质为字节通信流
                ？来源于非ASCII字节（如0xFF）无法显示
                属于正常现象，并非通信错误
    
    RESET行为异常定位（关键定位）
        实验现象：
            | 条件           | RESET行为        |
            | ------------ | -------------- |
            | USB 供电（串口连接） | RESET 可正常启动    |
            | 独立供电 + 串口连接  | RESET 无效，需断电重启 |
            | 独立供电 + 未连接串口 | RESET 正常       |
        分析结论：
            串口连接状态会影响MCU的复位启动行为
        进一步推断：
            板载CH340及其一键下载电路参与RESET/BOOT控制
            串口连接时，控制线状态可能干扰启动流程
            RESET ≠ 上电复位（POR），无法完全恢复系统初始状态
        工程结论：
            该问题属于板级行为特性，而非软件错误
