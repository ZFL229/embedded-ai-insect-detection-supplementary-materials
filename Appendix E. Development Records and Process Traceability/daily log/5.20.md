异常检测机制前移
    在重新梳理当前结构系统时，重新分析了：capture_len异常检测机制的位置问题。
    此前：
        当前机制位于：
            JPEG传输完成后
        即：
            MCU已经完成JPEG发送后
        才会检查：
            capture_len是否异常。
    该结构存在问题：
        即便图像已经发生：
            ISP状态污染；
            JPEG严重畸变；
        其仍然会进入：
            PC通信链路；
            图像保存链路；
            后续AI链路。
    因此今日正式决定：
        将capture_len异常检测机制前移至：
            图像采集完成后；
            图像传输开始前；
    当前系统正式形成：
        Capture → Frame Validation Gate → Transmission
    的结构。

    即，当前：
        capture_len
    不再属于：
        “事后统计”
    而正式成为：
        ”图像准入门禁机制“
示例：
void TriggerEvent_CaptureAndSend(void)
{
    /* Step 1: Capture image */
    if (CaptureImage() != OK)
    {
        LogStatus(CAPTURE_FAILED);
        /* Force rebuild acquisition chain */
        RebuildAcquisitionChain(FORCED_RECOVERY);
        return;
    }
    /* Step 2: Image health check */
    image_abnormal =
        (capture_len < MIN_VALID_CAPTURE_LEN);
    /* Step 3: Recovery mechanism */
    if (image_abnormal)
    {
        /* Forced recovery with Camera Power Cycle */
        RebuildAcquisitionChain(FORCED_RECOVERY);
        LogStatus(IMAGE_ABNORMAL);
        /* Discard abnormal image */
        return;
    }
    else
    {
        /* Lightweight recovery */
        RebuildAcquisitionChain(LIGHT_RECOVERY);
    }
    /* Step 4: Send image */
    if (SendImage() != OK)
    {
        LogStatus(TRANSFER_FAILED);
        ClearImageBuffer();
        return;
    }
    /* Step 5: Update image ID */
    image_id++;
}

异常状态机制重新梳理
    在完成异常检测机制前移后，重新梳理当前系统时，注意到此前实际已经存在：
        多条异常清理路径
    当前系统正式形成-三类异常状态清理机制：
        1. CAPTURE_FAILED 
        即： 
            图像采集阶段失败； 
            JPEG未生成； 
            capture_len == 0； 
        当前行为： 
            清空Buffer； 
            重建采集链路； 
            返回Idle状态。 
        2. IMAGE_ABNORMAL 
        即： 
            图像虽然成功采集； 
        但： 
            capture_len低于阈值； 
            ISP状态异常； 
            图像被判定为坏帧； 
        当前行为： 
            丢弃当前图像； 
            清空Buffer； 
        触发： 
            Soft Reset； 
            PWDN Recovery； 
            返回Idle状态。 
        3. TRANSFER_FAILED 
        即： 
            图像本身合法； 
        但： 
            PC READY失败； 
            UART发送失败； 
            ACK等待失败； 
            Session异常中断； 
        当前行为： 
            清空Buffer； 
            终止当前Session； 
            返回Idle状态。
