今日目标：
    完成摄像头图像采集闭环验证，确认MCU是否能够获取并输出一帧JPEG图像

完成工作
    摄像头初始化问题最终定位与解决
        在前一阶段中，系统已确认：
            SCCB通信正常
            摄像头ID读取正常
            PCLK存在告诉跳变
            VSYNC/HRED未被MCU识别
    问题表现为：
        DCMI始终TIMEOUT / DMA ERROR
        无法进入有效数据采集阶段
    问题本质分析:
        通过对比前一阶段调试结论，结合当前现象，重新定位问题：
            摄像头已输出数据(PCLK存在)
            但未进入“正确图像输出模式”
        即：
            摄像头在工作，但没有输出“标准DVP图像流”
    解决方案：
        昨日引入开源初始化流程(核心步骤)：
            OV2640_JPEG_INIT
            OV2640_YUV422
            OV2640_JPEG
            分辨率配置（320x240）
        并提供了工程参考：
            1.摄像头初始化流程(寄存器序列)
            2.图像输出模式配置(JPEG/YUV链路)
            3.DCMI ioc硬件配置参考(PCLK：Rising edge/VSYNC：Active Low/HSYNC：Active Low)
        该部分参考直接对齐了MCU对同步信号的解析方式
        工程意义：
            避免了大量:
                极性组合试错
                DCMI模式猜测
                同步信号误判
                寄存器序列盲猜
            大幅降低了调试复杂度与事件成本
    验证结果
        通过Debug变量观测：
            dbg_pclk_cnt  ≫ 0
            dbg_vsync_cnt ≫ 0
            dbg_href_cnt  ≫ 0
        结论：
            摄像头成功进入DVP图像输出模式
            VSYNC/HREF/PCLK全部有效
    
    DCMI + DMA单帧采集实现
        初始方案(失败)
            采用：
                Snapshot + FrameEventCallback（事件驱动）
            问题表现：
                cam_dbg.done = 0
                程序卡在等待函数，ST-LINK灯不断闪烁，证明一直在进行数据传输
        结论：
            FrameEventCallback在当前JPEG+Snapshot模式下不可靠
    
    策略调整
        参考开源项目，调整为：
            HAL_DCMI_Start_DMA(...)
            ↓
            HAL_Delay(固定时间)
            ↓
            HAL_DCMI_Suspend / Stop
        即：
            放弃事件驱动，采用时间驱动完成采集
    
    JPEG传输验证机制建立
        实现：
            扫描 buffer 中：
            FF D8 → SOI
            FF D9 → EOI
        Debug结果：
            soi_index = 0
            eoi_index ≈ 3240
        结论：
            成功获取完整JPEG数据流
    
    内存数据导出与图像验证
        通过GDB执行：
            dump binary memory test.jpg <addr> <addr+len>
        导出JPEG文件并在PC端打开。
        验证结果：
            图像可正常显示
            JPEG结构完成(FF D8 → FF D9)
            无明显数据损坏
