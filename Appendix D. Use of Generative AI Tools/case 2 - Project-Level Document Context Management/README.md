# case 2 - Project-Level Document Context Management - Project Knowledge Sources

## Usage

This workflow shows how project documents can be supplied to the AI as persistent reference material. The source collection contains stage summaries, a hardware reference manual, and other project records. These files give the AI background on earlier design decisions, hardware constraints, terminology, and the evolution of the system.

Typical use:

1. Upload or attach authoritative project documents as sources.
2. Use clear filenames and keep stage summaries separated by development phase.
3. Ask questions that require evidence from several stages or from the hardware manual.
4. Request drafting, comparison, or technical review based on the supplied sources.
5. Check important hardware details against the original manual before implementation.
